Reglas de mantenimiento

obligatorio
  • Documentar cualquier cron nuevo el mismo dia que se cree.
  • Incluir nombre, estado, frecuencia, funcion, entrega y job ID.
  • Indicar archivos, calendarios o repositorios afectados.
  • Si un cron falla, anotar error resumido y accion pendiente.
  • Actualizar este HTML cuando cambie `cron Job&skills.md`.
  • Para recordatorios puntuales, usar Google o WhatsApp exactamente a la hora; borrar los caducados.
